Under the Hood

The details, when you're ready for them.

Memory System

Project Instructions (APEX.md) — Drop a file with your project rules, architecture notes, coding conventions. Automatically injected into every session for every model.

Persistent Memory (MEMORY.md) — Accumulated knowledge that persists across sessions. The AI reads and updates memory as it learns about your work.

Semantic Search — All memory and conversation transcripts indexed with vector embeddings. Hybrid recall combines keyword matching with semantic similarity.

Whisper Injection — On each message, the server silently searches for relevant memories and injects them into the prompt. The AI "remembers" previous sessions without you asking.

Session Recovery — When sessions compact or the server restarts, a structured briefing is generated: what was discussed, what was decided, what's pending. Conversations survive crashes.

Cross-Model Continuity — All backends receive the same project instructions, memories, and whisper injections. Switch models mid-project — the new model already knows what you were working on.

Skills Engine

Built-in skills:

  • /recall — Full-text search across all conversation transcripts
  • /embedding — Semantic search across memory and transcripts
  • /codex — Delegate a task to Codex as a parallel background agent
  • /grok — Web research via xAI with live search
  • /first-principles — 4-layer deep analysis framework
  • /stop-slop — Score and rewrite AI-sounding text to sound human
  • /evaluate-repo — Sandbox-assess any GitHub repo

Build your own: Skills are just directories with a SKILL.md file. Apex discovers them automatically — no registry, no configuration. Two types: executable (runs a script) and thinking (AI follows instructions). Usage metrics and feedback tracked automatically. The /improve meta-skill analyzes usage and proposes improvements.

Admin Dashboard

Full web-based management portal at /admin. 61 REST API endpoints — usable by humans (web UI) and AI agents (JSON API).

  • Health — Server status, per-provider model reachability, database stats, TLS status
  • Configuration — Server settings, feature toggles, model defaults, timeouts
  • Credentials — Set/update API keys with masked input, format validation, audit logging
  • TLS Certificates — Generate CA/server/client certs, download .p12 bundles and QR codes
  • Workspace — Edit project instructions and memory files from the browser
  • Sessions — View active sessions, token usage, force compaction, live log streaming
Alert System

Multi-channel notifications for when the AI or your systems need to reach you:

  • In-app — Real-time WebSocket delivery with badge counters
  • Telegram — Bot delivery for mobile push notifications
  • REST API — Any external script, cron job, or service can POST alerts
  • Persistent inbox — All alerts stored with ack/unack tracking, searchable by category and severity

Alerts support custom categories, severity levels (info/warning/critical), and structured metadata.

Security
  • mTLS — Client certificate authentication for all connections. No passwords to steal.
  • Credential management — Masked input, format validation, rate limiting, audit logging
  • Atomic writes — Config and credential updates use temp file + rename. No partial state.
  • Input sanitization — Control character, null byte, and injection rejection on all inputs
  • Secrets isolation — Credentials in .env only, never in the database
  • TLS everywhere — HTTPS for all connections, with built-in cert generation tools
Architecture

No frameworks. No npm. No build step. The entire frontend is embedded in the Python file. One file to deploy, audit, and understand.

FAQ

Is this just an API wrapper?
No. Claude runs through the Agent SDK with full tool access (file read/write, bash, search). Codex runs with sandbox permissions. Local models get a custom tool-calling loop. It's closer to Claude Code than a chat interface.
How much does it cost to run?
If you already pay for Claude and/or ChatGPT, Apex adds zero cost for those models. Local models are free. Only Grok requires a separate API key. Hosting is your own hardware.
Can I use only local models?
Yes. Install Ollama, pull a model, start Apex. No API keys, no accounts, no internet needed for inference. You still get the full memory system, skills, and dashboard.
Can multiple people use one server?
The current architecture is single-user. Multi-user with RBAC is on the roadmap.
